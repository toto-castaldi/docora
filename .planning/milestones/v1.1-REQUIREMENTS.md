# Requirements: Docora v1.1 Polish & Resilience

**Defined:** 2026-02-14
**Core Value:** See failures before clients report them, and fix them with one click.

## v1.1 Requirements

All 13 requirements shipped and validated.

### Dashboard Cleanup

- [x] **DCLEAN-01**: Admin repositories page is removed from dashboard navigation and routes — Phase 6
- [x] **DCLEAN-02**: Backend admin API endpoints for global repository listing are removed — Phase 6

### Token Management

- [x] **TOKEN-01**: App can update GitHub token for a private repo via PATCH endpoint without losing delivery history — Phase 7
- [x] **TOKEN-02**: Token update validates against GitHub API before persisting — Phase 7
- [x] **TOKEN-03**: Token update resets error state (retry_count, last_error) for a fresh start — Phase 7

### Failure Notifications

- [x] **NOTIFY-01**: App receives a sync_failed webhook when circuit breaker opens for a watched repository — Phase 8
- [x] **NOTIFY-02**: Sync failure notification includes error type, message, retry count, and circuit breaker status — Phase 8
- [x] **NOTIFY-03**: Sync failure notification uses same HMAC authentication as file notifications — Phase 8

### Documentation Site

- [x] **DOCS-01**: Homepage explains what Docora is, its value proposition, and how it works — Phase 9
- [x] **DOCS-02**: API section documents onboard endpoint (POST /api/apps/onboard) with request/response schemas — Phase 9
- [x] **DOCS-03**: API section documents repository endpoints (POST /api/repositories, DELETE, PATCH) — Phase 9
- [x] **DOCS-04**: Webhook section documents all notification types sent to apps (create, update, delete, sync_failed) — Phase 9
- [x] **DOCS-05**: Site has clear navigation between homepage, API docs, and webhook docs — Phase 9
